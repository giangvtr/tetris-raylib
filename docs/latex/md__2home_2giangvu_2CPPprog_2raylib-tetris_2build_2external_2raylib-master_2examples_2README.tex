\begin{DoxyItemize}
\item {\ttfamily make} builds all examples
\item {\ttfamily make \mbox{[}module\mbox{]}} builds all examples for a particular module (e.\+g {\ttfamily make core})
\end{DoxyItemize}\hypertarget{md__2home_2giangvu_2CPPprog_2raylib-tetris_2build_2external_2raylib-master_2examples_2README_autotoc_md16}{}\doxysubsection{\texorpdfstring{With Zig}{With Zig}}\label{md__2home_2giangvu_2CPPprog_2raylib-tetris_2build_2external_2raylib-master_2examples_2README_autotoc_md16}
The \href{https://ziglang.org/}{\texttt{ Zig}} toolchain can compile {\ttfamily C} and {\ttfamily C++} in addition to {\ttfamily Zig}. You may find it easier to use than other toolchains, especially when it comes to cross-\/compiling.


\begin{DoxyItemize}
\item {\ttfamily zig build} to compile all examples
\item {\ttfamily zig build \mbox{[}module\mbox{]}} to compile all examples for a module (e.\+g. {\ttfamily zig build core})
\item {\ttfamily zig build \mbox{[}example\mbox{]}} to compile {\itshape and run} a particular example (e.\+g. {\ttfamily zig build core\+\_\+basic\+\_\+window})
