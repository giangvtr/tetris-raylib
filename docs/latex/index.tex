A simple cross platform template for setting up a project with the bleeding edge raylib code. Works with C or C++.\hypertarget{index_autotoc_md101}{}\doxysection{\texorpdfstring{Supported Platforms}{Supported Platforms}}\label{index_autotoc_md101}
Quickstart supports the main 3 desktop platforms\+:
\begin{DoxyItemize}
\item Windows
\item Linux
\item Mac\+OS
\end{DoxyItemize}\hypertarget{index_autotoc_md102}{}\doxysection{\texorpdfstring{Naming projects}{Naming projects}}\label{index_autotoc_md102}
Do not name your game project \textquotesingle{}raylib\textquotesingle{}, it will conflict with the raylib library.\hypertarget{index_autotoc_md103}{}\doxysection{\texorpdfstring{VSCode Users (all platforms)}{VSCode Users (all platforms)}}\label{index_autotoc_md103}
{\itshape Note} You must have a compiler toolchain installed in addition to vscode.


\begin{DoxyItemize}
\item Download the quickstart
\item Rename the folder to your game name
\item Open the folder in VSCode
\item Run the build task ( CTRL+\+SHIFT+B or F5 )
\item You are good to go
\end{DoxyItemize}\hypertarget{index_autotoc_md104}{}\doxysection{\texorpdfstring{Windows Users}{Windows Users}}\label{index_autotoc_md104}
There are two compiler toolchains available for windows, Min\+GW-\/\+W64 (a free compiler using GCC), and Microsoft Visual Studio \hypertarget{index_autotoc_md105}{}\doxysubsection{\texorpdfstring{Using Min\+GW-\/\+W64}{Using MinGW-W64}}\label{index_autotoc_md105}

\begin{DoxyItemize}
\item Double click the {\ttfamily build-\/\+Min\+GW-\/\+W64.\+bat} file
\item CD into the folder in your terminal
\item run {\ttfamily make}
\item You are good to go
\end{DoxyItemize}\hypertarget{index_autotoc_md106}{}\doxysubsubsection{\texorpdfstring{Note on Min\+GW-\/64 versions}{Note on MinGW-64 versions}}\label{index_autotoc_md106}
Make sure you have a modern version of Min\+GW-\/\+W64 (not mingw). The best place to get it is from the W64devkit from \href{https://github.com/skeeto/w64devkit/releases}{\texttt{ https\+://github.\+com/skeeto/w64devkit/releases}} or the version installed with the raylib installer \hypertarget{index_autotoc_md107}{}\doxyparagraph{\texorpdfstring{If you have installed raylib from the installer}{If you have installed raylib from the installer}}\label{index_autotoc_md107}
Make sure you have added the path

{\ttfamily C\+:\textbackslash{}raylib\textbackslash{}w64devkit\textbackslash{}bin}

To your path environment variable so that the compiler that came with raylib can be found.

DO NOT INSTALL ANOTHER Min\+GW-\/\+W64 from another source such as msys2, you don\textquotesingle{}t need it.\hypertarget{index_autotoc_md108}{}\doxysubsection{\texorpdfstring{Microsoft Visual Studio}{Microsoft Visual Studio}}\label{index_autotoc_md108}

\begin{DoxyItemize}
\item Run {\ttfamily build-\/\+Visual\+Studio2022.\+bat}
\item double click the {\ttfamily .sln} file that is generated
\item develop your game
\item you are good to go
\end{DoxyItemize}\hypertarget{index_autotoc_md109}{}\doxysection{\texorpdfstring{Linux Users}{Linux Users}}\label{index_autotoc_md109}

\begin{DoxyItemize}
\item CD into the build folder
\item run {\ttfamily ./premake5 gmake2}
\item CD back to the root
\item run {\ttfamily make}
\item you are good to go
\end{DoxyItemize}\hypertarget{index_autotoc_md110}{}\doxysection{\texorpdfstring{Mac\+OS Users}{MacOS Users}}\label{index_autotoc_md110}

\begin{DoxyItemize}
\item CD into the build folder
\item run {\ttfamily ./premake5.osx gmake2}
\item CD back to the root
\item run {\ttfamily make}
\item you are good to go
\end{DoxyItemize}\hypertarget{index_autotoc_md111}{}\doxysection{\texorpdfstring{Output files}{Output files}}\label{index_autotoc_md111}
The built code will be in the bin dir\hypertarget{index_autotoc_md112}{}\doxysection{\texorpdfstring{Working directories and the resources folder}{Working directories and the resources folder}}\label{index_autotoc_md112}
The example uses a utility function from {\ttfamily path\+\_\+utils.\+h} that will find the resources dir and set it as the current working directory. This is very useful when starting out. If you wish to manage your own working directory you can simply remove the call to the function and the header.\hypertarget{index_autotoc_md113}{}\doxysection{\texorpdfstring{Changing to C++}{Changing to C++}}\label{index_autotoc_md113}
Simply rename {\ttfamily src/main.\+c} to {\ttfamily src/main.\+cpp} and re-\/run the steps above and do a clean build.\hypertarget{index_autotoc_md114}{}\doxysection{\texorpdfstring{Using your own code}{Using your own code}}\label{index_autotoc_md114}
Simply remove {\ttfamily src/main.\+c} and replace it with your code, and re-\/run the steps above and do a clean build.\hypertarget{index_autotoc_md115}{}\doxysection{\texorpdfstring{Building for other Open\+GL targets}{Building for other OpenGL targets}}\label{index_autotoc_md115}
